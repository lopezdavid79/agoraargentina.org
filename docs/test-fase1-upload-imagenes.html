2. Upload manual desde el admin

Manual

Probá el flujo completo en el navegador.

2.1. Levantar el servidor

npm run dev

El servidor arranca en http://localhost:3000.

2.2. Iniciar sesión como admin

  • Andá a http://localhost:3000
  • Iniciá sesión con un usuario admin
  • Te debería llevar al dashboard (/admin/dashboard)

2.3. Crear una noticia con imagen

  1. Buscá el botón "Nueva Noticia" en el dashboard
  2. Completá titulo, copete, contenido
  3. Imagen: ahora aparece un <input type="file"> en vez del campo de URL
  4. Seleccioná un archivo JPEG o PNG menor a 5MB
  5. Verificá que se muestre un preview de la imagen antes de enviar
  6. Envíá el formulario
  7. ✅ Resultado esperado: redirige al dashboard sin errores. La noticia aparece en la tabla.

2.4. Verificar en Firestore (opcional)

Si tenés acceso a Firebase Console, revisá la colección noticias. El documento nuevo debería tener:

imagenUrl: "/images/noticias/{timestamp}-{nombre-del-archivo}"

2.5. Verificar el archivo en disco

ls public/images/noticias/

Debería aparecer el archivo subido con el nombre formateado: {timestamp}-{slug}.png.

2.6. Probar editar una noticia existente

  1. Hacé clic en "Editar" en una noticia existente
  2. Ahora el campo de imagen es un <input type="file">
  3. Si no seleccionás un archivo nuevo y guardás, la imagen existente se conserva
  4. Si seleccionás un archivo nuevo, se reemplaza

2.7. Crear / editar un curso

Mismo procedimiento que con noticias, pero los archivos se guardan en public/images/cursos/ y el campo Firestore es imagen (no imagenUrl).

3. Probar casos de error

Manual

Validaciones de seguridad.

3.1. Archivo demasiado grande

  • Seleccioná un archivo de más de 5MB
  • ✅ Resultado esperado: error 413. El servidor responde con un mensaje de error (flash).

3.2. Tipo de archivo no permitido

  • Seleccioná un archivo PDF, GIF, WebP o SVG
  • ✅ Resultado esperado: error 400 con mensaje "Tipo de archivo no permitido".

3.3. Crear noticia/curso sin imagen

  • Completá el formulario de nueva noticia sin seleccionar archivo
  • ✅ Resultado esperado: error 400 "Imagen requerida".
